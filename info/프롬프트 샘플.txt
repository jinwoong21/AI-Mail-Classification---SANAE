내용 :  1000자이내

위 형태를 갖는 json 데이터 샘플 30개 생성

2. 규칙 
출력 포맷은  info = [ {  title:__, content:__, to:__, from:__, sdate:___, gubun=0또는1, target=0~4중1개값,  rnow=0또는1} , ..] 와 같이 할 것

3.규칙
3-1) gubun은 0(일반적인 사적 이메일), 1(업무와 관련된 이메일)로 분류
3-2) target은 인사팀(0), 개발팀(1), 영업팀(2), 해외사업부(3), 교육사업부(4)로 구분하며 메일 내용은 해당 부서에 적합한 업무 내용으로 만들 것
3-3) gubun: 1(업무와 관련된 이메일) 중 target이 노출되지 않는 제목인 경우 내용을 통해 해당 부서를 예측할 수 있도록 content 생성할것
3-4) rnow: 일반(1), 긴급(1)
3-5) title은 인사팀 직원 채용 공고 관련,개발팀 버그 수정 요청 와 같이 target이 노출되는 제목과 노출되지 않는 제목으로 섞어서 생성할 것
3-6) 샘플 데이터는 gubun과 관련된 샘플 0(일반적인 사적 이메일), 1(업무와 관련된 이메일)을 3 : 7 비율로 생성할 것
3-7) 샘플 데이터는 target은 인사팀(0), 개발팀(1), 영업팀(2), 해외사업부(3), 교육사업부(4)로 구분하여 각 팀당 10건씩 생성하되 rnow: 일반(1), 긴급(1)을 7:3 비율로 생성할 것
3-8) 샘플 데이터의 content는  3-1) ~ 3-7)의 조건을 충실히 따를 것
